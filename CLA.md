## How to sign

When you open your first pull request to the AGPL'd portion of the Symdue codebase, the CLA Assistant bot will detect that you haven't signed and post a comment with a "sign the CLA" link. Click the link, sign in with GitHub OAuth, and the bot will mark your PR as CLA-cleared.

You only need to sign once. The signature is associated with your GitHub account and applies to all your future contributions.

## Why a CLA?

Symdue offers commercial licenses on top of the AGPL'd codebase (see [COMMERCIAL_LICENSE.md](COMMERCIAL_LICENSE.md)). To do this, Symdue needs the right to relicense contributed code under non-AGPL terms when selling commercial licenses to specific customers. This CLA grants Symdue that right.

This is the standard pattern for dual-licensed open source projects:
- [Grafana CLA](https://grafana.com/about/contribute/cla/) — same broad license grant
- [Sentry CLA](https://sentry.io/legal/contribution-agreement/) — same pattern
- [Cal.com CLA](https://github.com/calcom/cal.com/blob/main/CLA.md) — HarmonyCLA-derived
- [HashiCorp CLA](https://www.hashicorp.com/cla) — same pattern
- [Apache Foundation ICLA](https://www.apache.org/licenses/contributor-agreements.html) — the original template

If you have concerns about signing the CLA — for example, if your employer has restrictive contribution policies — please reach out before contributing. We'll work with you to find an arrangement (signing on behalf of the employer, contributing to the Apache 2.0 SDK / demos which don't require CLA, etc.).

## Lighter alternative for SDK + demo contributions: Developer Certificate of Origin (DCO)

Contributions to the Apache 2.0 plugin SDK, workflow JSON spec, and `demos/` directory do **not** require this CLA. Instead, you simply sign-off your commits with the standard [Developer Certificate of Origin (DCO)](https://developercertificate.org/):

```
Signed-off-by: Your Name <your.email@example.com>
```

The DCO is a lightweight alternative: you attest "I have the right to contribute this under the project's license," but you don't grant Symdue relicensing rights. This is sufficient for the Apache 2.0 portions of the codebase because Apache 2.0 itself permits commercial relicensing.
